Question 1

Start a personal website on GitHub Pages using a static site generator such as pelican or hugo. You can find tutorials for how to do on the web. This will be your data science project portfolio. The first project in your portfolio is to write a blog describing how you did this exercise.

Answer:

STEP 1. Installation

Install Hugo
brew install hugo
hugo version

Install Git
git --version

Step 2: Build the site locally

Set up the site
cd /Users/pohungniu
hugo new site Pohung8411

Choose a theme (here, I use coder)
cd Pohung8411
cd themes
git ninit
git submodule https://github.com/budparr/gohugo-theme-coder.git
echo theme = "coder" >> config.toml

After finishing this, we can code in the config.toml. But, I just use the code in exampleSite to build my website. I think it’s a better way to do the website.

Create the first blog
cd Pohung8411
hugo new about.md

I also use the code in exampleSite here. I think it’s more efficient.

Test the site locally
cd Pohung8411
hugo server

We can also go to http://localhost:1313/ in the web browser and the page is shown.

Step 3: Deploy to Git Pages

Get the website rendered in the public folder
cd /Users/pohungniu/Pohung8411
hugo

Create the Github Repository
a. Go to Github webpage and click on the “New” to create a new repository, contain the fully rendered version of the Hugo website.
b. then don’t initiate with a README.md, click on “Create Repository”.
My repository name here is Pohung8411.github.io

Go back to terminal and commit
cd public
git init
git add .
git remote add origin https://github.com/Pohung8411/Pohung8411.github.io.git
git commit -m “first commit”
git push origin master

And we can see our website by seeing the published link (https://pohung8411.github.io/ in my case)
