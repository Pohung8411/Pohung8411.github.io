(Fewer than 500,000 people: 489,359) Problem 5 Smallest multiple

2520 is the smallest number that can be divided by eah of the numbers from 1 to 10 without any remainder.
What is the smallest positive number that is evenly divisible by all of the numbers from 1 to 20?

Answer:

Result:

232,792,560  

Python Code:

def gcd(x, y):
          ‘‘‘finding the greatest common divisor of x and y

          Args:
                  x (int) : any integer
                  y (int) : any integer
                  The inputs that will be operated on by ‘gcd’

          Returns:
                  the greatest common divisor of x and y (integer)

          Example:
                 suppose that we input x = 20, y = 15, we know the y does equal to 0. So, we will return gcd (y, x % y), that is gcd (15, 5), and now our x = 15, y = 5. We then input x = 15, y = 5 into the gcd function. Again, y does not equal to 0. So, we will return gcd(y, x % y), that is gcd(5, 0), and now our x = 5, y = 0. We then input x = 5, y =0 into the gcd function. At this step, we figure out our y does equal to 0, so our gcd function will return us 5. As a result, 5 is the greatest common divisor of 20 and 15.

          ’’’
          #gcd(x, y)
          if y == 0:
                  return x
          else:
                  return gcd(y, x % y)

def lcm(x, y):
          ‘‘‘finding the least common multiple of x and y

          Args:
                  x (int) : any integer
                  y (int) : any integer
                  The inputs that will be operated on by ‘lcm’

          Returns:
                  the least common multiple of x and y (integer)

          Example:
                 Here, we just use the mathematical fact, the product of two integers equals the product of their greatest common divisor and least common multiple. So, we can find out that x and y’s least common multiple equals to their product divided by their greatest common factor. We can list an example to explain this. Suppose our x = 20 and y = 15. From above, we know their greatest common divisor is 5. So, we can easily find out that their least common multiple equals to (20*15) / 5 = 60. SO, their least common multiple equals to 60.

'’’
#lcm(x, y)
return x * y / gcd(x, y)

n = 1
for i in range(1, 20):
          n = lcm(n, i)
print(n)

Interpretation:

For the definition, we just write down functions for gcd and lcm. GCD is a function for finding the highest common factor. LCM is a function for finding the least common multiple.
The logic behind this code is that we start from n = 1 and try to run i in range from 1 to 21. To be specific, when i = 1, n = lcm(1, 1) = 1, so we print out 1 first. Then, when i = 2, n = lcm(1, 2) = 2, we print out 2. When i = 3, n = lcm(2, 3) = 6, we print out 6. When i = 4, n = lcm(4, 6) = 12, we print out 12. When i = 5, n = lcm(5, 12) = 60, we print out 60. By doing this repeately until i = 20, we can find out our answer 232, 792, 560. Actually, when we run the coding, it will print out all n during the process. 1

(Fewer than 100,000 people: 92,991) Problem 34 Digit factorials

145 is a curious number, as 1! + 4! + 5! = 1 + 24 + 120 = 145.
Find the sum of all numbers which are equal to the sum of the factorial of their digits.
Note: As 1! = 1 and 2! = 2 are not sums they are not included.

Answer:

Result:

40,730

Python Code:

first step

from math import factorial
import time
start = time.time()
f = [factorial(0),
factorial(1),
factorial(2),
factorial(3),
factorial(4),
factorial(5),
factorial(6),
factorial(7),
factorial(8),
factorial(9)]

second step

def factorial_digits(n):
          ‘‘‘finding the factorial of every one-digit in n

          Args:
                  n (int) : any integer
                  The inputs that will be operated on by ‘factorial_digits’

          Returns:
                  the factorial of every one-digit in n (integer)

          Example:
                 this might be a little abstract, so I will take an example. Suppose n is 359, what is the factorial of every one-digit in this number? We can use the function below. First, we set s to 0 and then we make s = s + f[359%10] = s + f[9]. Calculated by the built-in function, we can know what value of 9! is (362,880). Then, we let n = 359 // 10, which returns 35 as a result. Then, we return to the while function and find s = 362,880 + f[35%10] = 362,880 + f[5]. Calculated by the build-in function, we can know what value of 5! is (120). Then, we let n = 35 // 10, which returns 3 as a result. Again, we return to the while function and find s = 363,000 + f[3%10] = 363,000 + f[3]. Calculated by the build-in function, we can know what value of 3! is (6). Then, we let n =3 // 10, which returns 0 as a result. Then, this funciton will return our s number 362,880 + 120 + 6 = 363,006, which is the factorial of every one-digit in 359.

'’’
#factorial_digits(n)
s=0
while n:
                s += f[n%10] (equal to s = s + f[n%10])
                n //= 10 (equal to n = n // 10)
return s

third step

solution = 0
for i in range(10, 1854721):
          if factorial_digits(i) == i:
                  solution += i (equal to solution = solution + i)

final step

print(solution)
end = time.time()
print(end - start)

Interpretation:

For the first step, we just import the factorial function for calculating factorial and then import time module for calculating time of execution (Importing time is not necessary for finding the solution). The key part here is to find the factorials of numbers from 0-9. That is, 1!, 2!, 3!, 4!, 5!, 6!, 7!, 8!, 9!, which will be used later. We only need to find the factorials of one digit because the factorials of two digits are not needed in this question.

For the second step, we need to make a definition for factorial_digits(n). This step is quite simple. We set s = 0 at first and n is the number we want to test. The formulas of s = s + f[n%10] and n = n // 10 help us return the factorial of every one-digit in n. This return will then be listed as s.

For the third step, this might be a little tricky here. Setting solution = 0 is understandable. Setting the lower bound of 10 is also understandable because, except 1 and 2, the factorials of every one-digit do not equal to these one-digits themselves. However, we need to explain why we set our upper bound as 1,854,721 in this case. If you are interested in this, you can refer to Factorion Wikipedia. But, I will briefly talk about the concept here. The concept is that if n is a natural number of d digits that is a factorion, then 10^(d-1) <= n <= 9!d. This fails to hold for d>= 8, thus n has at most 7 digits, and the first upper bound we can find is 9,999,999. But the maximum sum of factorials of digits for a 7 digit number is 9!*7 = 2,540,160 establishing the second upper bound. Going further, 9!6 is 2,177,280, and the only 7 digit number not larger than 2,540,160 containing six 9’s is 1,999,999, which is not a factorion by inspection. The next highest sum would be given by 1,999,998, yielding a third upper bound of 1,854,721. So, we can set our lower bound of 10 and upper bound of 1,854,721. If i in this range equal to factorial_digits(i), then we will add these i into our solution.

For the final step, we just need to print our solution and we will get our result 40,730. The coding of print(end - start) is not necessary here, but it can help us know the total time of execution. For me, it is 349.1530570983887.

(Fewer than 25,000 people: 23,318) Problem 206 Concealed Square

Find the unique positive integer whose square has the form 1_2_3_4_5_6_7_8_9_0, where each “_” is a single digit.

Answer:

Result:

1389019170

Python Code:

def match(n):
          ‘‘‘defining the pattern 1_2_3_4_5_6_7_8_9 in our topic

          Args:
                  n (int) : any integer
                  The inputs that will be operated on by ‘match’

          Returns:
                  Boolean value (True or False)

          Example:
           We can just take 19293949596979899, the square of 138902663, as our example. Firstly, we let 19293949788491569 as string ‘19293949788491569’. Then, we need to figure out whether int(s[0]) == 1, int(s[2]) == 2, …., int(s[16]) == 9. If one of them does not, the boolean value will be False. In our example, it is False because int(s[8]) does not equal 5. And then we use return not function. This means if our answer is False, it will return True. if our answer is True, it will return False. So, if we input 19293949788491569, it will return True.

'’’
#Boolean value: True = not fit the pattern, False = fits the pattern
s = str(n)
return not all(int(s[x*2]) == x + 1 for x in range(9)))
(So, this definition just talks about the pattern in our topic 1_2_3_4_5_6_7_8_9)

n = 138,902,663 (integer part of sqrt(19293949596979899))
(this part talks about the maximum possible number’s round-down square root)

while match(n*n): n -= 2 (equal to : n = n - 2)
else: print ("Project Euler 206 Solution =", n*10)
(this part talks about we need to make our n*n fits the pattern 1_2_3_4_5_6_7_8_9. if match(n * n) returns true, which means some numbers do not fit the pattern, we will let n = n - 2. Until match(n * n) = False, which means all numbers fit the pattern, we print out our answer. Finally, don’t forget to add the number 0 behind this answer, which was found at the beginning of the practice.)

Interpretation:

First, we need to observe that the square of any number, n, will end in 00 if the last digit of n is 0. With this observation, we can fill out the last gap of the topic’s number. This number’s form would be 1_2_3_4_5_6_7_8_900.

Next, we can omit 00 and search for square root of 1_2_3_4_5_6_.7_8_9. We just need to add 0 after the solution we find at the end. For the number 1_2_3_4_5_6_.7_8_9, we can easily observe that the square of root for this number must be ending with 3 or 7, an odd number, and we can skip even-numbered n from consideration.

The final step is to search with the maximum possible number, 19293949596979899, at the beginning. While its square root is not an integer, we round down it. This round-down integer is the largest possible square root. Starting with an odd number we count down this integer by 2 until the square fits our pattern. The result is 1389019170.
